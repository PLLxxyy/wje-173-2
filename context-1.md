- **老人管理**：录入、编辑、删除老人信息（姓名、年龄、住址、健康状况、紧急联系人、自理能力等级、是否独居）
- **筛选功能**：按自理能力筛选，重点关注独居老人
- **统计报表**：
  - 老人总数、服务订单数、完成率、独居老人数、志愿者数、服务人员数
  - 各类服务需求占比排行（饼图）
  - 自理能力分布（饼图）
  - 月度服务订单趋势（折线图）
  - 月度服务完成率趋势（柱状图）
  - 志愿者活跃度排行（表格）

### 2. 家属功能 ([FamilyDashboard.jsx](file:///Users/yu/仓库/pdd-6-ti/20260614-create-2/wje-173/frontend/src/pages/FamilyDashboard.jsx))
- **绑定老人**：关联社区老人，填写关系
- **服务订单**：预约上门服务（家政保洁、代买代办、陪诊就医、康复护理），选择时间和备注
- **健康档案**：查看老人健康记录（血压、心率、血糖、用药等）
- **服务记录**：查看历史服务内容、时长、反馈
- **志愿者需求**：发布陪伴聊天或代办事务需求

### 3. 服务人员功能 ([WorkerDashboard.jsx](file:///Users/yu/仓库/pdd-6-ti/20260614-create-2/wje-173/frontend/src/pages/WorkerDashboard.jsx))
- **待接单列表**：查看可接订单，接单
- **我的订单**：查看已接订单，完成服务后填写服务记录（服务内容、时长、老人反馈）

### 4. 志愿者功能 ([VolunteerDashboard.jsx](file:///Users/yu/仓库/pdd-6-ti/20260614-create-2/wje-173/frontend/src/pages/VolunteerDashboard.jsx))
- **需求列表**：查看需要陪伴聊天或代办的老人需求，报名参加
- **我的报名**：查看报名记录和状态

## 数据库表结构 ([init-db.js](file:///Users/yu/仓库/pdd-6-ti/20260614-create-2/wje-173/backend/init-db.js))
- `users` - 用户表（管理员、家属、服务人员、志愿者）
- `elderly` - 老人信息表
- `family_bindings` - 家属老人绑定关系
- `service_types` - 服务类型表
- `service_orders` - 服务订单表
- `service_records` - 服务记录表
- `health_records` - 健康档案表
- `volunteer_demands` - 志愿者需求表
- `volunteer_signups` - 志愿者报名表

## API 接口 ([server.js](file:///Users/yu/仓库/pdd-6-ti/20260614-create-2/wje-173/backend/server.js))
- 认证：`POST /api/auth/register`、`POST /api/auth/login`
- 老人管理：`GET/POST/PUT/DELETE /api/elderly`
- 家属绑定：`GET/POST /api/family/bindings`
- 服务订单：`GET/POST /api/service-orders`、`PUT /api/service-orders/:id/accept`、`PUT /api/service-orders/:id/complete`
- 志愿者需求：`GET/POST /api/volunteer-demands`、`POST /api/volunteer-demands/:id/signup`
- 统计：`GET /api/admin/stats`

## 启动方式

服务已经在运行中：
- **前端**：http://localhost:3000
- **后端**：http://localhost:3001

如需重新启动：

```bash
# 1. 安装依赖
npm run install:all

# 2. 初始化数据库（只需运行一次）
cd backend && npm run init-db

# 3. 同时启动前后端
npm run dev

# 或分别启动
npm run server   # 后端
npm run client   # 前端
```
